Mercado Pago (PIX, Cartão e Boleto)

Dica: NUNCA deixe Access Token no front-end. Guarde no backend. Aqui no painel, salve apenas chaves públicas e flags.

ATIVO

Usada no front para inicializar checkout/SDK (não é segredo).

Seu backend escolhe o Access Token conforme o modo.

Correios (limitações)

Você já usa Correios com limite de 30kg e dimensão máx. 100cm. Aqui você pode definir quando o sistema deve bloquear Correios e mandar para frete pesado/manual.

Frete manual (por UF + Capital/Interior)

Você define o valor manual por estado e se é capital, interior ou ambos. Ideal para móveis/eletros quando não dá Correios.

Ativo UF Capital/Interior Peso mín (kg) Peso máx (kg) Maior dim máx (cm) Preço (R$) Prazo (dias) Ação
Como o sistema decide “Capital x Interior”?
No checkout, dá pra usar o CEP (ViaCEP) e comparar com a capital do UF. Se a cidade do CEP for a capital, aplica “Capital”, senão “Interior”.
